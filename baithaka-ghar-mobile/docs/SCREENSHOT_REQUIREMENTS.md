# App Store Screenshot Requirements

## iOS App Store Screenshots

### Required Sizes

#### iPhone
- **6.7" Display** (iPhone 14 Pro Max, 15 Pro Max): 1290 x 2796 pixels
- **6.5" Display** (iPhone 11 Pro Max, XS Max): 1242 x 2688 pixels
- **5.5" Display** (iPhone 8 Plus): 1242 x 2208 pixels

#### iPad
- **12.9" Display** (iPad Pro): 2048 x 2732 pixels
- **11" Display** (iPad Pro): 1668 x 2388 pixels

### Screenshot Content Plan (5-10 screenshots)

1. **Home Screen - Property Listings**
   - Show diverse property cards with images
   - Display search bar at top
   - Show filters button
   - Include bottom navigation
   - Caption: "Discover Authentic Stays Across Nepal & India"

2. **Property Details**
   - Large property image gallery
   - Property name, location, rating
   - Amenities icons visible
   - Price clearly shown
   - "Book Now" button
   - Caption: "Detailed Property Information with Photos & Reviews"

3. **Booking Flow**
   - Date picker showing check-in/check-out
   - Guest selector
   - Price breakdown visible
   - "Confirm Booking" button
   - Caption: "Easy Booking in Just a Few Taps"

4. **Secure Payment**
   - Razorpay payment screen
   - Multiple payment options shown
   - Security badge/SSL indicator
   - Total amount prominent
   - Caption: "Secure Payments Powered by Razorpay"

5. **Booking Confirmation**
   - Success checkmark/animation
   - Booking details summary
   - Property image
   - "View Booking" and "Download Invoice" buttons
   - Caption: "Instant Confirmation & Digital Invoices"

6. **Messages Screen**
   - Chat interface with property owner
   - Profile pictures visible
   - Online status indicator
   - Typing indicator
   - Caption: "Direct Communication with Property Owners"

7. **Profile & Bookings**
   - User profile section
   - List of upcoming/past bookings
   - Favorite properties
   - Settings options
   - Caption: "Manage All Your Bookings in One Place"

8. **Reviews & Ratings** (Optional)
   - Review form with star rating
   - Photo upload option
   - Sample reviews visible
   - Caption: "Share Your Experience & Help Fellow Travelers"

9. **Map View** (Optional)
   - Interactive map with property pins
   - Property cards on bottom sheet
   - Location details
   - Caption: "Explore Properties on Interactive Map"

10. **Notifications** (Optional)
    - List of push notifications
    - Booking reminders
    - Special offers
    - Caption: "Stay Updated with Real-Time Notifications"

### Best Practices
- Use actual app interface (not mockups)
- Show diverse properties (hotels, homestays, etc.)
- Include realistic booking data
- Use high-quality, clear images
- Avoid text-heavy screenshots
- Show key features in first 3 screenshots
- Use device frames (optional but recommended)
- Maintain consistent branding colors

## Google Play Store Screenshots

### Required Sizes

#### Phone
- **Minimum**: 320px to 3840px (recommended: 1080 x 1920 pixels)
- **Maximum**: 3840px
- **Aspect Ratio**: 16:9 or 9:16

#### 7-inch Tablet
- **Recommended**: 1200 x 1920 pixels

#### 10-inch Tablet
- **Recommended**: 1600 x 2560 pixels

### Screenshot Content Plan (Minimum 2, Maximum 8)

Use the same content plan as iOS, but optimize for Android:

1. **Home Screen** - Property discovery
2. **Property Details** - Comprehensive information
3. **Booking Flow** - Simple 3-step process
4. **Payment** - Secure Razorpay integration
5. **Confirmation** - Instant confirmation & invoices
6. **Messaging** - Direct communication
7. **Profile** - Booking management
8. **Reviews** - Community feedback

### Best Practices
- First 2 screenshots are most important
- Use JPG or 24-bit PNG (no alpha)
- Maximum file size: 8MB per screenshot
- Show actual app UI
- Use Material Design guidelines
- Test on various screen sizes
- Include captions/text overlays (optional)

## App Icon

### iOS
- **App Store**: 1024 x 1024 pixels (PNG, no transparency)
- **iPhone**: Multiple sizes auto-generated by Xcode
- **iPad**: Multiple sizes auto-generated by Xcode

### Android
- **Play Store**: 512 x 512 pixels (PNG with transparency)
- **Adaptive Icon**:
  - Foreground: 108 x 108 dp (432 x 432 px @ xxxhdpi)
  - Background: 108 x 108 dp (432 x 432 px @ xxxhdpi)

### Design Guidelines
- Simple, recognizable design
- Works well at all sizes
- Represents the brand (house/home theme)
- No text or complex details
- Consistent with app's color scheme (#1a1a1a)

## App Preview Video (Optional but Recommended)

### iOS Specifications
- **Duration**: 15-30 seconds
- **Format**: .mov, .m4v, or .mp4
- **Resolution**: Same as screenshot sizes
- **Orientation**: Portrait
- **Audio**: Optional

### Android Specifications
- **Duration**: 30 seconds to 2 minutes
- **Format**: MPEG, AVI, WMV, 3GPP, MOV, MP4
- **Resolution**: Minimum 1280 x 720 pixels
- **File Size**: Maximum 100MB

### Video Storyboard

1. **0-2s**: App icon animation and splash screen
2. **2-5s**: Browse properties (scroll through listings)
3. **5-8s**: View property details (swipe through photos)
4. **8-11s**: Select dates and book
5. **11-13s**: Secure payment with Razorpay
6. **13-15s**: Booking confirmation celebration
7. **15-17s**: Chat with property owner
8. **17-19s**: Leave a review
9. **19-22s**: View profile and bookings
10. **22-25s**: Receive push notification
11. **25-30s**: End screen with app name, tagline, and download CTA

### Production Tips
- Use screen recording software (QuickTime, OBS)
- Record on actual device for authenticity
- Use smooth transitions between screens
- Add subtle background music (royalty-free)
- Include on-screen text for key features
- Show finger taps/gestures
- Keep it fast-paced and engaging
- End with strong call-to-action

## Screenshot Checklist

### Before Taking Screenshots
- [ ] Use production build (not development)
- [ ] Clear all debug logs and developer tools
- [ ] Use realistic, high-quality property images
- [ ] Create sample bookings with realistic data
- [ ] Ensure all text is readable
- [ ] Check for typos and grammatical errors
- [ ] Use actual device (not simulator) if possible
- [ ] Ensure proper lighting and colors

### Content Guidelines
- [ ] No personal/sensitive information visible
- [ ] No placeholder text (Lorem ipsum)
- [ ] No broken images or missing data
- [ ] Consistent branding throughout
- [ ] Professional property images
- [ ] Realistic prices and dates
- [ ] Proper grammar and spelling
- [ ] Cultural sensitivity (diverse properties/users)

### Technical Requirements
- [ ] Correct resolution for each platform
- [ ] Proper file format (PNG for iOS, JPG/PNG for Android)
- [ ] File size within limits
- [ ] No status bar issues (clock, battery, etc.)
- [ ] Proper orientation (portrait for most)
- [ ] No UI glitches or cutoffs
- [ ] High pixel density (2x or 3x)

### Localization (Future)
- [ ] Prepare screenshots for each language
- [ ] Translate on-screen text
- [ ] Use appropriate property examples per region
- [ ] Consider cultural preferences
